Project Overview

A back-end management system with RESTful api to keep track with every family's income and expense.

Tech Stack


Major Technologies: Python, Django, RESTful API, MySQL, Locust, Pytest, JWT Auth

Django includes an admin page serving as a service desk, aiding users in managing safety concerns and sensitive deletions. Built on Django's Rest Framework and JWT authentication, the platform ensures secure user access. Unauthorized users are restricted from modifying records, with data isolation preventing users from accessing other family's information. ORM, Python's automated and performance test (Locust) guarantee smooth performance under high user loads while minimize errors.

Complete User Mannual

Base URL (MUST be refered to when sending http requests):

Same as the current page's URL. As a reference: https://moneytracker-prod-9941b1d428c3.herokuapp.com.

User Login and Authentication:

  1. Create a user through authentication endpoint by entering first name, last name, password and email by appending the following to the base url: /auth/users/.

  2. After doing so, generate access token through /jwt/create/. Remember to save the token locally through front-end (eg. LocalStorage in Javascript) as it will be expired after a period of time (30 days for development sake).

    3. IMPORTANT: Unauthorized users have no access to any of the following endpoints for operating on future data, so make sure you authenticate yourself; authorized users can only access their own family & individual's incomes, expense and net income after linking themselves to their specific family to ensure the safety. Otherwise, 404 unauthorized or 403 forbidden response will be returned.

Family Creation:

Along with your JWT authentication header, you are autherized to create a family through base url + /tracker/families/ endpoint. Input the family name for your family and a password. The password here is not for accessing a family, but for our service desk to check your identity. Instead, a family code will be generated and that will be used for future secure access and management of your own family.

Linking Members to Family:

  1. Before linking the user with the family, we need to extract the user's user id by entering base url + /tracker/my-profile/me/.

  2. After a member is created, head over to base url + /tracker/families/::family_id/ endpoint to view the family profile or rename the family. Once checked, head over to base url + /tracker/families/::family_id/members/ endpoint; enter your member id in order to link yourself to the family.

    3. Note: Unlinking members is also possible by sending a POST request via base url + /tracker/families/::family_id/members/:: member_id/unlink-member/.

Operating Family Member's Earnings & Expense:

  1. To view earnings, expenses and net or post earnings and expenses: append the base url by /tracker/families/::family_id/members/:: member_id/earnings/. Change earnings/ to expense/ or records/ to view the rest. To post new earnings or expenses, simply follow the instruction and hit POST. However, we do not allow POST to records endpoint directly. NOTE: Sorting is allowed through filter button entering year and month to get a monthly report.

  2. To update or delete an earning or expense: append the base url by /tracker/families/::family_id/members/:: member_id/earnings/::earning_id/. Similar than before, earnings can be substituded by expenses. However, the record is one entity which does not come with record_id. Delete the earnings or expense by hitting sending the delete request on the same url.

  3. To view the net income of this individual: append the base url by /tracker/families/::family_id/members/:: member_id/records/. NOTE: Sorting is allowed by appending querystrings to the current url. (eg. ?earning__received_date__month=9 &expense__paid_date__month=6)

Viewing Family Records:

Similar to viewing individual member's record, it is also possible to view a list of all member's earnings, expense in a family. The BEAUTY here is that the family net income is calculated down below and can be filtered by entering years and months. Just start by appending the base url by tracker/families/::family_id/records/

IMPORTANT: Again, all the info related to family or individual earnings, expense, and the net are hidden from people who have not linked themselves to your family. Don't worry about the security issue as we have taken care of that for you.
