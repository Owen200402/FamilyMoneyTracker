Project Overview

A back-end management system with RESTful api to keep track with every family's income and expenses.

Tech Stack


Major Technologies: Python, Django, RESTful API, MySQL, Locust, Pytest, JWT Auth

Built on Django's Rest Framework and JWT authentication, the platform ensures secure user access. Unauthorized users are restricted from endpoints, with data isolation preventing users from accessing other family's information. Django ORM helps manipulate on the database server, while automated and performance test guarantee smooth performance under high user loads while minimize errors.

Complete Mannual For Developers

Base URL (MUST be refered to when sending http requests):

It is the same as the current page's URL: https://moneytracker-prod-9941b1d428c3.herokuapp.com.

User Login and Authentication:

  1. Create a user through authentication endpoint by entering first name, last name, password and email by appending the following to the base url: /auth/users/.

  2. After doing so, generate access token through /jwt/create/. Remember to save the token locally using front-end technologies like LocalStorage in Javascript.

    3. IMPORTANT: Unauthorized users cannot access certain endpoints. Authorized users can only access their own family and individual data.

Family Creation:

Along with your JWT authentication header, you can create a family by navigating to /tracker/families/ endpoint. Input the family name for your family and a password. The password here is not for accessing a family, but for our service desk to check your identity. Instead, a family code will be generated and that will be used for future secure access and management of your own family.

Linking Members to Family:

  1. Before linking the user with the family, we need to extract the user's user id by navigating to /tracker/my-profile/me/.

  2. After a member is created, head over to /tracker/families/::family_id/ endpoint to view the family profile or rename the family. Once checked, navigate to /tracker/families/::family_id/members/ endpoint; enter your member id in order to link yourself to the family.

    3. Note: Unlinking members is also possible by sending a POST request via endpoint /tracker/families/::family_id/members/:: member_id/unlink-member/.

Operating Family Member's Earnings & Expenses:

  1. To view earnings, expenses and net or post earnings and expenses: append the base url by /tracker/families/::family_id/members/:: member_id/earnings/. Change earnings/ to expenses/ or records/ to view the rest. To post new earnings or expenses, simply follow the instruction and hit POST. However, we do not allow POST to records endpoint directly. NOTE: Sorting is allowed through filter button entering year and month to get a monthly report.

  2. To update or delete an earning or expenses, navigate to /tracker/families/::family_id/members/:: member_id/earnings/::earning_id/. Similar than before, earnings can be substituded by expenses. However, the record is one entity which does not come with record_id. Delete the earnings or expenses by hitting sending the delete request on the same url.

  3. To view the net income of this individual: append the base url by /tracker/families/::family_id/members/:: member_id/records/. NOTE: Sorting is allowed by appending querystrings to the current url. (eg. ?earning__received_date__month=9 &expense__paid_date__month=6)

Viewing Family Records:

Similar to viewing individual member's record, it is also possible to view a list of all member's earnings, expenses in a family. The BEAUTY here is that the family net income is calculated down below and can be filtered by entering years and months. Just navigate to tracker/families/::family_id/records/.

IMPORTANT: All the info related to family or individual earnings, expenses, and the net are hidden from people who have not linked themselves to your family. Don't worry about the security issue as it has been taken care of that for you.
